# Advanced Chat Tabs

Самостоятельный клиентский мод для **Minecraft Java Edition 1.21.11** на Fabric. Он добавляет вкладки чата, фильтры, отдельные окна, индивидуальные и глобальные параметры, профили серверов и необязательное сохранение истории. LabyMod и его исходники, ресурсы или элементы брендинга не используются.

## Матрица версий

| Компонент | Зафиксированная версия |
|---|---:|
| Minecraft | `1.21.11` |
| Java | `21` |
| Fabric Loader | `0.19.3` |
| Fabric API | `0.141.4+1.21.11` |
| Yarn mappings | `1.21.11+build.6` |
| Fabric Loom | `1.15.5` |
| Gradle | `9.4.1` |
| JUnit Jupiter | `5.12.2` |

Все версии вынесены в `gradle.properties`. Проект использует Yarn, поскольку 1.21.11 является последней веткой Minecraft, для которой Fabric продолжил выпуск Yarn/Intermediary.

## Архитектура

Поток сообщения устроен так:

1. `ChatHudMixin` перехватывает финальную ванильную перегрузку `ChatHud#addMessage(Text, MessageSignatureData, MessageIndicator)`.
2. `ChatTabManager` создаёт неизменяемую внутреннюю модель `ChatMessage<VanillaChatPayload>` и сохраняет исходный `Text`, подпись и индикатор доверия.
3. `FilterEngine` проверяет заранее кэшируемые условия. Регулярные выражения компилируются при первом использовании и не выполняются каждый кадр.
4. `ChatMessageRouter` строит `RoutePlan`: вкладки назначения, локальное или глобальное скрытие, визуальные действия и счётчики.
5. Для каждой вкладки хранится отдельный ванильный `ChatHud.ChatState`. Благодаря этому активная вкладка использует стандартный HUD Minecraft и сохраняет click/hover-компоненты, ссылки, прокрутку и подписи живых сообщений.
6. `HistoryStore` и `ConfigManager` выполняют отложенную атомарную запись через временный файл и резервную копию.
7. `HudElementRegistry` рисует плавающие окна после ванильного слоя чата.

Основные области ответственности:

- `chat`: модель сообщения, история, объединение и маршрутизация;
- `filter`: условия, действия, безопасная проверка regex и тестовый результат;
- `tab`: вкладки, настройки и управление ванильными `ChatState`;
- `window`: модели, размещение, перемещение, изменение размера и рендер окон;
- `config`: глобальный профиль, серверные профили, миграции и история;
- `screen`: собственные экраны вкладок, фильтров, действий и окон;
- `render`: панель вкладок и клиентские слои;
- `mixin`: три точечных клиентских внедрения;
- `command` и `input`: клиентские команды и горячие клавиши.

Полное дерево лежит в `PROJECT_TREE.txt`.

## Реализованные возможности

- автоматическая системная вкладка с именем сервера или мира;
- пользовательские вкладки с UUID, порядком, закреплением, историей и непрочитанными сообщениями;
- создание, переименование, дублирование, очистка, перемещение и удаление вкладок;
- переключение мышью, колёсиком и горячими клавишами;
- фильтры `contains`, `starts with`, `ends with`, `exact`, `regex`;
- логика ANY/ALL, исключающие условия, регистр, hover-текст, отправитель, тип и статус доверия;
- локальное и глобальное скрытие;
- добавление в текущую или выбранную вкладку;
- цвет текста, метка важности, префикс-значок, цвет/альфа фона для плавающих окон;
- звук с громкостью, высотой тона, cooldown и предварительным прослушиванием;
- action bar через отдельную узкую точку внедрения;
- объединение последовательных одинаковых сообщений с интервалом и счётчиком;
- защита каждой вкладки от ванильной очистки чата;
- временные метки с пользовательским `DateTimeFormatter`-шаблоном;
- глобальные или индивидуальные настройки;
- профили по адресу сервера и отдельный профиль одиночной игры;
- атомарное сохранение JSON, `.bak`, версия схемы и миграция;
- необязательная ограниченная история между запусками;
- плавающие окна с перемещением, изменением размера, прозрачностью, сворачиванием, закреплением, привязкой к краям и ограничением экраном;
- английская и русская локализации;
- клиентские команды `/act`;
- unit-тесты ядра.

## Сборка

Требуется JDK 21 и доступ к Maven Central, Fabric Maven и Gradle Distributions.

Linux/macOS:

```bash
chmod +x gradlew
./gradlew build
```

Windows:

```bat
gradlew.bat build
```

Готовый remapped JAR появится в `build/libs/advanced-chat-tabs-1.0.0.jar`.

Скрипты `gradlew` и `gradlew.bat` загружают зафиксированный Gradle 9.4.1 в локальную папку `.gradle-bootstrap/`, после чего выполняют обычную Loom-сборку. Бинарный `gradle-wrapper.jar` намеренно не вложен в исходный архив.

Дополнительные команды:

```bash
./gradlew test
./gradlew runClient
./gradlew clean build
```

## Установка

1. Установить Fabric Loader `0.19.3` для Minecraft `1.21.11`.
2. Поместить Fabric API `0.141.4+1.21.11` в папку `mods`.
3. Собрать проект и поместить `advanced-chat-tabs-1.0.0.jar` в ту же папку `mods`.
4. Запустить клиент Minecraft с Java 21.
5. Открыть меню командой `/act`.

Мод имеет `environment: client` и не требуется на сервере.

## Команды

- `/act` — главное меню;
- `/act tabs` — список вкладок;
- `/act filters` — фильтры активной вкладки;
- `/act windows` — редактор окон;
- `/act reload` — перечитать JSON и заново привязать текущий профиль;
- `/act debug` — переключить отладочный режим.

Команды регистрируются через Fabric Client Command API и не отправляются серверу.

## Горячие клавиши

В меню управления Minecraft появляется категория `Advanced Chat Tabs`:

- следующая вкладка;
- предыдущая вкладка;
- настройки активной вкладки;
- новая вкладка;
- показать или скрыть окна.

Назначения можно изменить стандартным экраном управления Minecraft.

## Конфигурация

После первого сохранения создаётся:

```text
config/advanced_chat_tabs/
├── global.json
├── global.json.bak
├── servers.json
├── servers.json.bak
├── windows.json
├── windows.json.bak
└── histories/
    └── <uuid-профиля>/
        └── <uuid-вкладки>.json
```

Запись откладывается на 750 мс после изменения, история на 1500 мс. Файл сначала записывается как `.tmp`, текущая версия копируется в `.bak`, затем выполняется атомарное перемещение, если файловая система его поддерживает. Gson игнорирует неизвестные поля.

Адрес сервера нормализуется в ключ профиля. Одиночная игра получает ключ на основе имени мира. Токены, данные аккаунта, ключи и сетевые пакеты не сохраняются.

## Примеры фильтров

### Личные сообщения

- название: `ЛС`;
- должно включать: `[ЛС]`;
- режим: `содержит`;
- действие: добавить в текущую вкладку;
- пример: `[ЛС] Player123: привет`.

### Убрать приглашение голосового чата

- должно включать: `голосового чата`;
- не должно включать: оставить пустым;
- действие: `скрыть сообщение`;
- глобальное скрытие: включено;
- пример: `Для голосового чата установите Simple Voice Chat.`

### События администрации

- регулярное выражение: `^Администратор.*событие$`;
- действия: пометить важным, фон `#AA5A1A1A`, звук `minecraft:block.note_block.pling`;
- пример: `Администратор запустил событие`.

### Торговля

- должно включать: `Продам`, `алмазы`;
- логический режим: `все условия`;
- действие: добавить в вкладку `Торговля`;
- пример: `Продам алмазы`.

### Вход игроков

- должно включать: `Игрок`, `вошёл на сервер`;
- логический режим: `все условия`;
- объединение одинаковых сообщений: включено;
- интервал: `10000` мс;
- пример: `Игрок вошёл на сервер`.

В редакторе фильтра поле «Проверить сообщение» показывает совпадение, сработавшее условие, исключение, ошибку regex и список действий. Некорректный шаблон возвращает диагностическое сообщение и не выбрасывает исключение наружу.

## Mixins

### `ChatHudMixin`

Внедряется в финальную перегрузку `addMessage(Text, MessageSignatureData, MessageIndicator)` на `HEAD`. Это последняя общая точка перед добавлением в HUD, поэтому мод получает итоговый `Text`, уже сформированный Minecraft или другим модом. Сетевые пакеты не отменяются и подписи не подделываются.

Второе внедрение находится в `clear(boolean)`. Мод поочерёдно очищает только состояния вкладок с отключённой защитой и возвращает активный `ChatState`. Никакие несвязанные пакеты не перехватываются.

### `ChatScreenMixin`

На `TAIL` метода `render` рисует интерактивную панель вкладок. В начале `mouseClicked(Click, boolean)` и `mouseScrolled(double, double, double, double)` обрабатывает только попадания в панель, после чего оставляет ванильное поведение без изменений.

### `InGameHudMixin`

В начале `setOverlayMessage(Text, boolean)` зеркалирует action bar в общий маршрут. Ванильный action bar остаётся видимым, кроме случая явного глобального скрытия фильтром.

## Производительность

- фильтры запускаются только при получении сообщения;
- скомпилированные `Pattern` кэшируются по параметрам фильтра;
- истории имеют жёсткий предел;
- конфигурация и история сохраняются с debounce;
- плавающий рендер берёт только последние видимые строки;
- удалённые окна удаляются из конфигурационной коллекции;
- рендер не выполняет regex и не перечитывает JSON.

## Известные ограничения текущей реализации

1. В этой среде не удалось выполнить настоящую Loom-сборку: изолированный контейнер не разрешает DNS для `services.gradle.org`. Результаты локальной проверки описаны в `VALIDATION.md`. Перед публикацией JAR необходимо выполнить `./gradlew clean build` на машине с интернетом и запустить `runClient`.
2. Живые сообщения активной вкладки сохраняют исходные click/hover-компоненты, стили, подпись и ванильную прокрутку. История, восстановленная после перезапуска, хранится как безопасный обычный текст без кликов, hover и криптографической подписи.
3. Кликабельность и hover плавающих окон пока не реализованы. Они остаются у активной ванильной вкладки.
4. Цвет и альфа фона действий фильтра отображаются в плавающих окнах. Встроенный фон строк активного `ChatHud` не заменяется отдельным цветом для каждой строки.
5. Параметры масштаба, межстрочного интервала, времени показа и плавного исчезновения сохранены в модели и редакторе, но активная ванильная вкладка продолжает использовать соответствующие глобальные параметры Minecraft.
6. Определение отправителя и части типов сообщений в общей точке `ChatHud` эвристическое. Action bar определяется точно отдельным hook, подпись и системный статус сохраняются по ванильным аргументам.
7. Порядок фильтров меняется кнопками вверх/вниз, а не drag-and-drop.
8. Первое и последнее время повтора сохраняются в истории, но отдельная hover-подсказка на счётчике `×N` ещё не добавлена.
9. Выбор цветов реализован через ARGB/RGB-поле и предварительный образец, а не через полноэкранную палитру.
10. Редактор окон поддерживает перемещение и изменение размера, но переключение нескольких вкладок внутри одного плавающего окна остаётся базовым: новое окно создаётся с активной вкладкой.

Эти ограничения перечислены прямо, потому что слово «полностью» не является компилятором и не должно использоваться вместо проверки игры.
